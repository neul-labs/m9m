    Trigger Nodes¶

    Trigger nodes start workflow execution based on events, schedules, or manual activation.

    Start Node¶

    Manual trigger for testing and development.

    {
  "type": "n8n-nodes-base.start",
  "parameters": {}
}

    Execute manually via CLI: 

    m9m execute workflow.json

    Webhook Node¶

    HTTP endpoint that triggers workflows.

    Basic Webhook¶

    {
  "type": "n8n-nodes-base.webhook",
  "parameters": {
    "path": "my-webhook",
    "httpMethod": "POST"
  }
}

    Webhook URL: https://your-m9m-host/webhook/my-webhook

    With Authentication¶

    {
  "parameters": {
    "path": "secure-webhook",
    "httpMethod": "POST",
    "authentication": "headerAuth",
    "headerName": "X-API-Key",
    "headerValue": "={{ $env.WEBHOOK_SECRET }}"
  }
}

    Response Configuration¶

    {
  "parameters": {
    "path": "webhook",
    "httpMethod": "POST",
    "responseMode": "responseNode",
    "responseData": "allEntries"
  }
}

    HTTP Methods¶

    Method Use Case
    GET Retrieve data, health checks
    POST Submit data, most common
    PUT Update resources
    DELETE Remove resources

    Query Parameters¶

    Access query parameters: 

    {{ $json.query.paramName }}

    Request Body¶

    Access request body: 

    {{ $json.body.fieldName }}

    Headers¶

    Access headers: 

    {{ $json.headers['content-type'] }}

    Cron Node¶

    Schedule-based triggers.

    Every Minute¶

    {
  "type": "n8n-nodes-base.cron",
  "parameters": {
    "triggerTimes": {
      "item": [
        {"mode": "everyMinute"}
      ]
    }
  }
}

    Every Hour¶

    {
  "parameters": {
    "triggerTimes": {
      "item": [
        {"mode": "everyHour", "minute": 0}
      ]
    }
  }
}

    Daily at Specific Time¶

    {
  "parameters": {
    "triggerTimes": {
      "item": [
        {
          "mode": "everyDay",
          "hour": 9,
          "minute": 0
        }
      ]
    }
  }
}

    Weekly¶

    {
  "parameters": {
    "triggerTimes": {
      "item": [
        {
          "mode": "everyWeek",
          "weekday": "monday",
          "hour": 9,
          "minute": 0
        }
      ]
    }
  }
}

    Monthly¶

    {
  "parameters": {
    "triggerTimes": {
      "item": [
        {
          "mode": "everyMonth",
          "dayOfMonth": 1,
          "hour": 0,
          "minute": 0
        }
      ]
    }
  }
}

    Cron Expression¶

    For complex schedules:

    {
  "parameters": {
    "triggerTimes": {
      "item": [
        {
          "mode": "custom",
          "cronExpression": "0 */15 9-17 * * MON-FRI"
        }
      ]
    }
  }
}

    Cron format: second minute hour day-of-month month day-of-week

    Timezone¶

    {
  "parameters": {
    "triggerTimes": {
      "item": [{"mode": "everyDay", "hour": 9, "minute": 0}]
    },
    "timezone": "America/New_York"
  }
}

    Interval Node¶

    Fixed interval triggers.

    {
  "type": "n8n-nodes-base.interval",
  "parameters": {
    "interval": 5,
    "unit": "minutes"
  }
}

    Units¶

    • seconds
    • minutes
    • hours

    Error Trigger¶

    Triggers when another workflow fails.

    {
  "type": "n8n-nodes-base.errorTrigger",
  "parameters": {}
}

    Receives error data: 

    {{ $json.workflow.name }}
{{ $json.error.message }}
{{ $json.execution.id }}

    Manual Trigger¶

    Explicit trigger for testing.

    {
  "type": "n8n-nodes-base.manualTrigger",
  "parameters": {}
}

    Service-Specific Triggers¶

    Slack Event¶

    {
  "type": "n8n-nodes-base.slackTrigger",
  "parameters": {
    "events": ["message.channels", "reaction_added"]
  },
  "credentials": {
    "slackApi": {"id": "1", "name": "Slack"}
  }
}

    GitHub Webhook¶

    {
  "type": "n8n-nodes-base.githubTrigger",
  "parameters": {
    "events": ["push", "pull_request"]
  },
  "credentials": {
    "githubApi": {"id": "1", "name": "GitHub"}
  }
}

    Email (IMAP)¶

    {
  "type": "n8n-nodes-base.imapTrigger",
  "parameters": {
    "mailbox": "INBOX",
    "action": "read"
  },
  "credentials": {
    "imap": {"id": "1", "name": "Email"}
  }
}

    Multiple Triggers¶

    A workflow can have multiple triggers:

    {
  "nodes": [
    {
      "id": "cron-trigger",
      "type": "n8n-nodes-base.cron",
      "parameters": {
        "triggerTimes": {"item": [{"mode": "everyHour"}]}
      }
    },
    {
      "id": "webhook-trigger",
      "type": "n8n-nodes-base.webhook",
      "parameters": {
        "path": "manual-run",
        "httpMethod": "POST"
      }
    },
    {
      "id": "process",
      "type": "n8n-nodes-base.set"
    }
  ],
  "connections": {
    "cron-trigger": {
      "main": [[{"node": "process", "type": "main", "index": 0}]]
    },
    "webhook-trigger": {
      "main": [[{"node": "process", "type": "main", "index": 0}]]
    }
  }
}

    Trigger Configuration¶

    Activate/Deactivate¶

    Triggers only run when the workflow is active:

    # Activate workflow
m9m workflow activate wf-001

# Deactivate
m9m workflow deactivate wf-001

    Test Triggers¶

    Test without activating:

    m9m execute workflow.json --trigger-data '{"body": {"test": true}}'

    Webhook Security¶

    HMAC Signature¶

    Verify webhook signatures:

    {
  "parameters": {
    "path": "secure",
    "httpMethod": "POST",
    "authentication": "hmacSignature",
    "signatureHeader": "X-Hub-Signature-256",
    "algorithm": "sha256"
  }
}

    IP Whitelist¶

    Configure in server settings:

    webhooks:
  allowedIps:
    - "192.168.1.0/24"
    - "10.0.0.1"

    Rate Limiting¶

    webhooks:
  rateLimit:
    requests: 100
    period: "1m"

    Best Practices¶

    1. Use meaningful webhook paths - /orders/new not /webhook1
    2. Secure webhooks with authentication
    3. Set appropriate cron intervals - don't poll too frequently
    4. Handle duplicate events - webhooks may retry
    5. Log trigger events for debugging
